Overview

The frontend of the website is accessed via dentonbible.org. The backend is available at dentonbible.org/wp-admin. The backend is where all changes to the site are made.

Registered users can be given access to various parts of the website to add/edit content, change the appearance, upload files, and more.

The parent or root site, dentonbible.org, is where the main pages are. These pages are used to give detail about general church information as well as high-level descriptions of individual ministries.

Ministries that need much more than a single page can offer are given access to a sub-site. Ministries like these include CollegeLife, Men's, Women's, Marriage & Family, and so on. A sub-site is an entirely new website located within a subdomain (i.e. men.dentonbible.org). However, it can be managed by a handfull of Super Admins that have access to the root site as well as all sub-sites. All content resides within a single database.

Each sub-site can use the same theme as the root site. They also have the ability of using entirely different themes.

Web Splashes

A "Web Splash" is used to highlight an upcoming event, major news, or anything needing a spotlight on the home page. Each splash is required to have an image.

How to add a web splash

  1. From the main menu, click Posts and then Add New.
  2. In the Title field, enter a relevant title.
  3. Add any descriptive text into the editor.
  4. To add an image (more info):
    1. Click the media button.
    2. Select an image to upload. Images must be 636px x 187px or will otherwise be automatically cropped.
    3. After the image is finished uploading, edit the title if necessary and click Save all changes.
  5. Select the "Featured" category.
  6. Optionally choose an expiration date/time from the "Post Expirator" box.
  7. Optionally turn off the ability for visitorst to comment from the "Discussion" box.
  8. Click Publish or edit the publish date/time to schedule the post to publish at a later time.

How to remove a web splash

Web splashes can be assigned an expiration date. However, in the case that a date is not assigned or the splash needs to be removed early, simply change the Post Status from "Published" to "Draft". You can also delete the splash.

Publications

A new content type in version 0.2 is a "publication". At its onset, publications are "First Cup", "Common Ground", and "Starting Point". Any number of other publications can be added.

How to add a new publication type

  1. From the main menu, click Publications and then Publication Type.
  2. Give the new publication type a Name
    1. (Optional) Give the publication a unique slug. It will automatically be generated from the Name.
    2. (Optional) You can assign publciation in a hierarchy.
    3. (Optional) You can give a description of the publication type.
  3. Click Add New Publication Type.

How to add a new publication

  1. From the main menu, click Publications and then Add New.
  2. Click Select Files and choose the publication and thumbnail you'd like to upload.
    1. After the image and publciation are finished uploading click Save all changes.
  3. Title the post anything. But for simplicty's sake, you may want to use: {publication name} {publication date} (i.e. First Cup January 1, 2010).
  4. Check the Publication Type that is associated with the publication you uploaded.
  5. Change the date of the post to that of the publication. For example, publishing associates the post with the current day and time. Instead, change that to the date the publication was released (or will be released).
  6. Click Publish.

Child Themes

A WordPress child theme is a theme that inherits the functionality of another theme, called the parent theme, and allows you to modify, or add to, the functionality of that parent theme.

The main theme, dbc, is the parent.
